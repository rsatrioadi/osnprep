Bacaan · Sesi 3

Anatomi Pseudokode — Membaca Kode Seperti Membaca Resep

Apa yang Dimaksud "Menelusuri Kode"?

Bayangkan kamu menerima sebuah resep masakan. Kamu tidak perlu masuk ke dapur untuk mengetahui apa hasilnya — kamu bisa membacanya langkah demi langkah dan membayangkan apa yang terjadi pada setiap bahan. Menelusuri kode (code tracing) adalah hal yang persis sama: kamu membaca instruksi yang ditulis dalam pseudokode atau C++, lalu kamu mencatat dengan tangan apa yang terjadi pada setiap baris — nilai variabel berapa, kondisi mana yang terpenuhi, apa yang dikembalikan — tanpa menjalankan program di komputer.

Di OSN-K, Bagian C hampir seluruhnya adalah soal penelusuran kode. Kamu tidak perlu bisa menulis kode, apalagi menjalankannya. Yang kamu butuhkan adalah kemampuan membaca dan mengikuti logika kode secara akurat, satu langkah pada satu waktu.

Elemen Dasar yang Perlu Kamu Kenali

Ada lima elemen yang muncul berulang kali di kode-kode OSN-K. Kamu tidak perlu menghafal sintaksnya dengan sempurna; yang penting adalah kamu mengerti apa yang dilakukan setiap elemen ketika kamu menemuinya.

1. Variabel dan tipe data. Variabel adalah "kotak" bernama yang menyimpan satu nilai pada satu waktu. Setiap variabel memiliki tipe yang menentukan jenis nilai yang boleh disimpannya. Tiga tipe yang paling sering muncul di OSN-K adalah int (bilangan bulat), bool (nilai benar/salah — ditulis true atau false), array (sekumpulan elemen bertipe data sama), dan string (teks, yang terkadang dapat dipandang sebagai array of karakter). Ketika menelusuri, tugas utamamu adalah mencatat nilai setiap variabel di setiap langkah. Jangan andalkan ingatan; tuliskan.

int n = 5; // variabel n menyimpan bilangan bulat, nilai awal 5 bool hasil = true; // variabel hasil menyimpan nilai benar/salah int numbers[] = {10, 20, 30}; // array dengan 3 elemen bilangan bulat string S = "AKU"; // variabel S menyimpan teks, nilai awal "AKU"

Tipe data lainnya antara lain float dan double (nilai real seperti 3.1415), dan char (satu karakter seperti 'a').

2. Ekspresi dan operator. Ekspresi adalah kombinasi nilai, variabel, dan operator yang menghasilkan satu nilai baru. Operator yang sering muncul: aritmetika (+ - * /), sisa bagi (%), perbandingan (== != < > <= >=), dan logika (&& untuk AND, || untuk OR). Ketika menelusuri, evaluasi setiap ekspresi dengan nilai variabel yang berlaku saat itu, bukan nilai awal.

n + 1 // ekspresi aritmetika: jika n=5, hasilnya 6 n % 3 == 0 // ekspresi perbandingan: apakah n habis dibagi 3? i > 0 && i < 10 // ekspresi logika: apakah i antara 1 dan 9? numbers[0] // akses elemen array: indeks dimulai dari 0; hasilnya 10

Mirip di matematika, operator punya urutan prioritas (preseden). Tabel berikut merangkum operator yang relevan untuk OSN-K, dari prioritas tertinggi ke terendah:

3. Percabangan (if–else). Memilih jalur yang akan dijalankan berdasarkan hasil evaluasi suatu ekspresi. Jika ekspresi bernilai true, blok if dijalankan; jika false, blok else dijalankan (jika ada). Gunakan nilai variabel yang berlaku saat baris itu dieksekusi, bukan nilai awal.

if (S[i] > S[i+1]) { // blok ini dijalankan jika karakter ke-i lebih besar dari ke-(i+1) }

Percabangan bisa berantai dengan else if untuk menangani lebih dari dua kasus:

// Konversi nilai 0–100 ke huruf A–E char grade(int nilai) { if (nilai >= 80) return 'A'; else if (nilai >= 65) return 'B'; else if (nilai >= 55) return 'C'; else if (nilai >= 40) return 'D'; else return 'E'; }

Perhatikan urutan: kondisi diperiksa dari atas ke bawah, dan fungsi berhenti di return pertama yang terpenuhi. Jika nilai = 72, kondisi pertama (72 >= 80) gagal, kondisi kedua (72 >= 65) terpenuhi → kembalikan 'B'. Kondisi ketiga dan seterusnya tidak pernah diperiksa.

4. Perulangan (for / while). Menjalankan satu blok instruksi berulang kali selama suatu ekspresi kondisi bernilai true. Cara menelusurinya: buat satu baris di tabelmu untuk setiap iterasi. Catat nilai semua variabel yang berubah — termasuk yang kelihatannya tidak berubah — agar tabelmu lengkap.

for (int i = 0; i < 4; i++) { // blok ini dijalankan 4 kali, untuk i = 0, 1, 2, 3 } while (a > 0) { // blok ini dijalankan selama a lebih dari 0 // akan berulang selamanya jika nilai a tidak pernah diganti di dalam blok }

5. Fungsi dan nilai kembalian. Fungsi adalah blok kode bernama yang menerima masukan (parameter), menjalankan serangkaian instruksi, dan mengeluarkan satu hasil (nilai kembalian) dengan kata kunci return. Ketika kode mencapai return, fungsi langsung berhenti — tidak ada baris sesudahnya yang dijalankan. Tipe kembalian ditulis sebelum nama fungsi: bool lebih(int a, int b) berarti fungsi bernama lebih yang menerima dua bilangan bulat dan mengembalikan benar/salah. Di OSN-K Bagian C, pertanyaannya hampir selalu: "Apa nilai kembalian fungsi ini untuk masukan tertentu?"

bool lebih(int a, int b) { // definisi fungsi lebih return (a >= b); // mengembalikan true jika a tidak lebih kecil dari b } lebih(5, 4) // pemanggilan fungsi dengan a=5 dan b=4, hasilnya true
💡 Catatan nama fungsi di OSN
Dalam pemrograman nyata, nama variabel dan fungsi harus mencerminkan kegunaannya. Di kompetisi seperti OSN, nama-nama tersebut sering disamarkan (seperti "cek", "foo", dll.) agar peserta tidak dapat menebak jawaban dari namanya.

Teknik Tabel Variabel

Alat paling penting dalam menelusuri kode iteratif adalah tabel variabel: sebuah tabel yang mencatat nilai setiap variabel penting setelah setiap iterasi. Tanpa tabel ini, penelusuran lebih dari 3 langkah akan hampir selalu berakhir dengan kesalahan.

⚠️ Aturan Tabel Variabel
Isi setiap sel — bahkan jika nilainya sama dengan baris sebelumnya. Konsistensi ini yang mencegah kesalahan.

Demo: Fungsi cek() — OSN-K 2024, Soal 26–28

bool cek(string S) { int n = S.length(); for (int i = 0; i < n-1; i++) { if (S[i] > S[i+1]) return false; // <-- early return! } return true; }
📋 Serangan Lima Langkah
L1 Baca cermat: Ada perulangan yang memeriksa pasangan karakter berturutan. Ada early return di tengah loop dan return setelah loop.
L2 Identifikasi tipe: Fungsi iteratif dengan early return — prediksi nilai kembalian TRUE/FALSE.
L3 Kasus kecil: cek("AB"): A < B → tidak melanggar → TRUE. cek("BA"): B > A → return false → FALSE.
L4 Invariant: TRUE jika dan hanya jika S tersusun tidak menurun: S[0] ≤ S[1] ≤ … ≤ S[n-1].
L5 Kasus tepi: S panjang 1? Loop tidak jalan (i < 0 tidak terpenuhi) → return true. Karakter sama (AAB)? A ≤ A → tidak melanggar → TRUE.
🔢 Penelusuran Langkah demi Langkah: cek("BEGITU")

String "BEGITU" = [B, E, G, I, T, U]. Loop berjalan untuk i = 0 sampai i = 4 (kondisi: i < n-1 = 5).

Iterasi i S[i] S[i+1] S[i] > S[i+1]? Yang terjadi
0 / 6 baris
✅ cek("BEGITU") = TRUE
Semua 5 pasang lolos — tidak ada penurunan alfabet. Loop selesai tanpa return false, lalu return true dieksekusi.

Bandingkan: cek("BEGINI") = FALSE karena N → I: 'N' > 'I' → return false langsung di i=4.

Telusuri Fungsi hitung()

Ini adalah fungsi buatan — bukan dari arsip OSN-K — yang dirancang sederhana agar kamu bisa fokus pada teknik tabel, bukan kerumitan logika.

int hitung(int N) { int hasil = 0; for (int i = 1; i <= N; i++) { if (N % i == 0) { hasil = hasil + 1; } } return hasil; }
💡 Catatan: N % i
N % i adalah sisa pembagian N dibagi i. Contoh: 10 % 3 = 1 karena 10 = 3×3 + 1. Jika N % i == 0, artinya i adalah pembagi N (membagi N tanpa sisa).

Rekursi — Menelusuri Tumpukan Panggilan

Apa Itu Rekursi?

Rekursi adalah ketika sebuah fungsi memanggil dirinya sendiri sebagai bagian dari penghitungannya. Ini sangat mirip dengan cara kita mendefinisikan beberapa hal dalam kehidupan sehari-hari — "untuk menghitung total biaya dari A ke Z, hitung dulu dari A ke kota sebelum Z, lalu tambahkan biaya terakhir." Definisi ini menyebut dirinya sendiri — itulah rekursi.

Rekursi ada di mana-mana, bahkan di luar pemrograman:

Efek Droste
Gambar yang mengandung dirinya sendiri — persis seperti fungsi yang memanggil dirinya sendiri.
Google "recursion"
Google menyarankan "Maksud kamu: recursion?" — menunjuk ke dirinya sendiri.
La Vache qui rit
Sapi memakai anting berbentuk logo yang sama — referensi diri tanpa akhir.
Segitiga Sierpiński
Setiap level = tiga salinan level sebelumnya. Definisi rekursif yang sempurna.

Fungsi rekursif selalu punya dua bagian yang wajib ada:

  • Basis (base case): kondisi berhenti. Jika terpenuhi, fungsi langsung mengembalikan nilai tanpa memanggil dirinya lagi.
  • Kasus rekursif: kondisi di mana fungsi memanggil dirinya sendiri dengan argumen yang "lebih kecil" atau "lebih dekat ke basis".

Contoh klasik: fungsi faktorial. Faktorial dari n (ditulis n!) didefinisikan sebagai hasil perkalian semua bilangan bulat positif dari 1 sampai n. Misalnya, 4! = 4 × 3 × 2 × 1 = 24. Perhatikan bahwa 4! = 4 × 3! — inilah inti dari definisi rekursifnya: untuk menghitung faktorial n, kita cukup tahu faktorial dari n−1, lalu kalikan dengan n.

int faktorial(int n) { if (n == 0) return 1; // BASIS: 0! = 1 return n * faktorial(n - 1); // n! = n × (n-1)! } // Cara berpikir: // faktorial(4) = 4 × faktorial(3) // = 4 × 3 × faktorial(2) // = 4 × 3 × 2 × faktorial(1) // = 4 × 3 × 2 × 1 × faktorial(0) // = 4 × 3 × 2 × 1 × 1 ← basis tercapai, nilai mengalir balik ke atas // = 24

Cara Menelusuri Rekursi: Tabel Bingkai

Setiap kali fungsi dipanggil, ia membuat bingkai (frame) tersendiri di memori. Ketika menelusuri rekursi, alat yang kamu butuhkan bukan tabel iterasi, melainkan tabel bingkai:

  • Setiap baris adalah satu panggilan fungsi (bukan satu iterasi).
  • Isi nilai kembalian dari bawah ke atas — bingkai yang pertama kali selesai adalah bingkai basis (yang paling bawah).
  • Setiap bingkai memiliki salinan variabelnya sendiri.
⚠️ Jebakan Paling Umum
Mengisi nilai kembalian dari atas ke bawah (dari bingkai pertama ke terakhir). Ini terbalik. Bingkai pertama (P1) tidak bisa diselesaikan karena ia menunggu hasil dari bingkai di bawahnya. Selalu mulai dari bingkai basis.

Demo: Fungsi panas() — OSN-K 2024, Soal 35–37

int panas(int X) { if (X == 0) return 0; // BASIS else return (X % 10) + panas(X / 10); // digit terakhir + rekursi }
💡 Dua Operasi Kunci
X % 10 = digit terakhir X. Contoh: 123 % 10 = 3.
X / 10 = X tanpa digit terakhir (pembagian bulat). Contoh: 123 / 10 = 12.
🌀 Penelusuran Tabel Bingkai: panas(123)

Fase 1 (langkah 1–4): gambar semua bingkai dari atas ke bawah. Fase 2 (langkah 5–8): isi nilai kembalian dari bawah ke atas, mulai dari basis.

Bingkai X Basis? Ekspresi Nilai Kembalian
0 / 8 langkah
✅ panas(123) = 6 = 1+2+3
Nilai mengalir dari bawah ke atas: P4→0, P3→1+0=1, P2→2+1=3, P1→3+3=6.
Invariant: panas(X) menghitung jumlah semua digit X. Verifikasi: panas(321) = 3+2+1 = 6.

Error Log — Catatan Kesalahan Pribadi

Setiap kali kamu mengerjakan soal dan hasilnya salah (atau benar tapi tidak yakin), catat satu baris di Error Log.

✅ Mengapa Error Log Penting?
Error Log bukan hukuman. Ini adalah alat diagnosis: semakin baik kamu memahami MENGAPA kamu salah, semakin cepat kamu bisa memperbaikinya. Tanpa Error Log, kamu akan terus membuat kesalahan yang sama tanpa sadar.
⏰ Penting
Isi Error Log ini sekarang, selagi ingatan masih segar. Isi minimal 1 baris berdasarkan soal yang kamu kerjakan di sesi ini. Jika tidak ada yang salah, tulis langkah mana yang paling sulit atau memakan waktu paling lama.

Ringkasan · Sesi 3

Yang Dipelajari Hari Ini

  • Menelusuri kode adalah membaca logika secara akurat satu langkah pada satu waktu — bukan menulis atau menjalankan kode.
  • Alat untuk perulangan: tabel variabel — satu baris per iterasi, isi semua kolom termasuk yang tidak berubah.
  • Alat untuk rekursi: tabel bingkai — satu baris per panggilan fungsi, isi nilai kembalian dari bawah ke atas (mulai dari basis).
  • Basis rekursi adalah bingkai yang pertama kali selesai, bukan yang pertama kali dipanggil.
  • Serangan Lima Langkah: baca cermat kondisi basis dan early return — keduanya adalah titik keluar yang menentukan seluruh alur fungsi.
